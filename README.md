# DiVA — **Di**scord **V**irtual **A**ssistant

A Claude-powered Discord bot with natural-language reminders, an evolving personality system, multi-model routing (Haiku + Sonnet), per-server credit billing, admin tools with confirmation/undo, per-channel memory, and per-user secretary mode for daily digests.

> DiVA can appear under a different nickname on any server via Discord's built-in "Change Nickname" feature. The global bot identity is DiVA, but individual servers can rename it (e.g., "kurbot").

## Features

### AI Chat
- **Multi-model routing** — short/casual messages route to Claude Haiku (cheap), complex queries and admin requests use Sonnet for quality
- **Reply chain awareness** — walks up to 3 messages in a reply thread for full context, including images from any user in the chain
- **Per-channel memory** — remembers the last 5 conversations in each channel
- **Conversation context** — pulls recent user history from the database for direct @mentions (not just replies)
- **Image analysis** — sees and describes images shared by users (paused in saving mode)
- **Web search** — searches the web for current events and recent info when needed

### Reminders (natural language)
- **Set:** _"@DiVA remind me about groceries in 3 hours"_ — auto-executes, no confirmation
- **Recurring:** _"every Monday at 9am"_ or _"every day at 8pm"_ — daily/weekly only, requires `/secretary on` for delivery setup
- **Cancel / reschedule:** _"cancel my groceries reminder"_, _"move it to 10pm"_ — confirmation card before execute
- **List:** `/reminder list` — position-based IDs (`#1, #2, #3`) that re-number as you cancel
- **AI-suggested titles** — Haiku proposes a polished version (e.g. _"groceries"_ → _"Grocery Shopping"_); a ✨ button lets you opt in
- **Sub-second precision** for short reminders ("in 1 minute") via `seconds_from_now`
- **Discord auto-timestamps** — every fire time renders in the viewer's locale and timezone
- **Restart-safe** — pending reminders survive bot restarts; in-memory timer Map is rebuilt from SQLite on boot

### Secretary Mode
- `/secretary on` — interactive setup wizard:
  1. Pick delivery: DM / current channel / pick another channel
  2. Optionally enable a daily digest at a chosen time
- Daily digest summarizes today's reminders + the next 48 hours
- Greeting line generated by Haiku (~$0.001/digest)
- `/secretary off` — disables the daily digest only; recurring reminders keep firing
- `/secretary status` — show current preferences
- `/secretary clear` — wipe all preferences (with confirmation)

### Per-Server Credit Billing
- Replaces the global monthly budget with per-server lifetime credits
- Each server's spend tracked separately in `guild_credits.total_spent_usd`
- Out-of-credits flow: bot replies once with a "top up to continue" notice, then pauses AI replies (with 24h cooldown so it doesn't spam)
- `/budget` shows per-server lifetime, spent, remaining, last-7-days trend
- `/credits show` — anyone can view balance + recent transactions
- `/credits add <guild_id> <amount>` — owner-only top-up
- Owner-managed flag bypasses out-of-credits checks (for your own servers)
- Every API call charged at model-aware pricing (Haiku $1/$5, Sonnet $3/$15 per M tokens)
- Saving mode at 85% — disables web search and image analysis

### Evolving Personality
- Every 15 interactions per server, a "dream digest" runs (lightweight Haiku call) — compresses recent conversations into a personality prompt (max 500 chars)
- The bot adapts to each server's tone, humor, slang, and vibe over time
- `/personality view` and `/personality reset` for admins

### Admin Tools (via Natural Language)
Admins can manage the server by chatting:

- **Channels** — create (text/voice/announcement/stage/forum), edit, move, set permissions
- **Roles** — create, edit, assign/remove
- **Server** — name, description, AFK settings, system channel
- **Threads** — create, archive, lock
- **Emojis** — create from URL, rename
- **Events** — schedule (voice/stage/external)
- **Members** — set/clear nicknames
- **Info** — list channels, list roles

Write operations show a ✅/❌ confirmation card. After execution, an ↩️ Undo button appears for 5 minutes (persisted to SQLite — survives restarts). Destructive actions (deleting channels/roles, kicking, banning) are blocked by policy.

### Per-Server Notification Channels
Each server admin can route notifications to channels of their choice:

- `/channel set kind:error #foo` — error alerts go here (in addition to the global owner channel)
- `/channel set kind:metrics #bar` — opt into per-guild weekly metrics report (Sundays 9 PM PT)
- `/channel set kind:notices #baz` — version update announcements
- `/channel show` — view current routing
- `/notices on/off` — toggle update announcements (default ON, opt-OUT)

### Update Notices
- On every version bump, posts a changelog to each server's notices channel (or system channel as fallback)
- Idempotent — posts once per version per server, never spams
- Skipped entirely on dev (`BOT_ENV !== 'production'`)

### Per-User Settings
- `/timezone <IANA zone>` or natural language _"set my timezone to Tokyo"_
- Each user's timezone, delivery preference, and digest time are independent
- Multi-user safe: position-based reminder IDs are computed per-user, ownership enforced on every cancel/reschedule

### Monitoring & Operations
- Weekly metrics summary to the owner's channel (global) + per-guild reports for opted-in servers
- Error notifications to a private Discord channel with full context
- Daily SQLite backups with 10-day retention (local only on Hobby Railway)
- Structured JSON logging with configurable levels
- Graceful shutdown on SIGTERM/SIGINT (DB closed cleanly, WAL checkpointed)
- Dev/prod separation — `BOT_ENV` env var gates update notices and DB filename (`bot.dev.db` vs `bot.db`)

## Prerequisites

- Node.js 20+
- A Discord bot application with **Message Content** and **Server Members** intents enabled
- An Anthropic API key

## Setup

```bash
git clone https://github.com/Jackie-Who/DiVA-Discord-Virtual-Assistant.git
cd DiVA-Discord-Virtual-Assistant
npm install

# For local dev: copy .env.dev.example to .env and fill in dev Discord app creds
cp .env.dev.example .env

# Register slash commands (one-time, idempotent)
node src/commands/register.js

# Start
npm start
```

### Dev/Prod Separation

DiVA distinguishes between dev and production via the `BOT_ENV` env var:

- **Production** (`BOT_ENV=production`): uses `bot.db`, posts update notices, charges real Anthropic API to your account
- **Development** (`BOT_ENV=development` or unset): uses `bot.dev.db`, suppresses update notices, includes a startup warning

The recommended workflow is to register a **separate Discord application** for development (e.g. "DiVA Dev") and use that token locally — your prod bot stays untouched while you iterate.

See `.env.dev.example` and `.env.prod.example` for templates.

### With PM2 (process manager)

```bash
npm install -g pm2
pm2 start ecosystem.config.js
pm2 startup
pm2 save
```

### With Docker / Railway

```bash
docker build -t diva .
docker run --env-file .env -v /path/to/data:/app/data diva
```

For Railway: set `DATA_DIR=/app/data` and attach a persistent volume at `/app/data`. Set `BOT_ENV=production` in service variables.

DiVA needs:

- Send Messages, Read Message History, Add Reactions
- Manage Channels, Manage Roles, Manage Messages
- Manage Emojis, Manage Events, Manage Nicknames
- Use Slash Commands

Invite URL (replace `YOUR_CLIENT_ID`):
```
https://discord.com/api/oauth2/authorize?client_id=YOUR_CLIENT_ID&permissions=1507328214086&scope=bot%20applications.commands
```

## Slash Commands

| Command | Who | Description |
|---|---|---|
| `/budget` | Anyone | Show this server's credit balance + recent spend |
| `/credits show` | Anyone | Same as `/budget` plus transaction log |
| `/credits add <guild_id> <amount>` | Bot owner | Top up a server's credits |
| `/timezone <zone>` | Anyone | Set your timezone (or just chat: _"set my timezone to Vancouver"_) |
| `/reminder list` | Anyone | List your active reminders (numbered #1..#N) |
| `/reminder delete <id>` | Anyone | Cancel a reminder by position |
| `/secretary on` | Anyone | Set up delivery preferences + (optional) daily digest |
| `/secretary off` | Anyone | Turn off the daily digest only (recurring reminders keep working) |
| `/secretary status` | Anyone | Show your current preferences |
| `/secretary clear` | Anyone | Wipe all your preferences (with confirmation) |
| `/channel set <kind> #channel` | Manage Server | Route errors / metrics / notices to a channel |
| `/channel show` | Manage Server | Show current channel routing |
| `/notices on` | Manage Server | Enable update notices (default) |
| `/notices off` | Manage Server | Disable update notices for this server |
| `/personality view` | Administrator | View the bot's evolved personality |
| `/personality reset` | Administrator | Reset the personality to start fresh |

> Reminder creation, cancellation, rescheduling, and most admin actions happen via natural language (@mention the bot) — slash commands are the visual management surface.

## Configuration

All configuration via environment variables. See `.env.dev.example` and `.env.prod.example` for templates.

| Variable | Default | Description |
|---|---|---|
| `DISCORD_TOKEN` | *required* | Discord bot token |
| `DISCORD_CLIENT_ID` | *required* | Discord application client ID |
| `DISCORD_GUILD_IDS` | *required* | Comma-separated guild IDs to register commands in |
| `ANTHROPIC_API_KEY` | *required* | Anthropic API key |
| `BOT_ENV` | `development` | `production` or `development`. Controls DB filename + update notices. |
| `OWNER_USER_ID` | | Discord user ID — gates owner-only commands like `/credits add` |
| `DATA_DIR` | project root | DB + backup directory (set to `/app/data` on Railway) |
| `ERROR_CHANNEL_ID` | | Global owner channel for error alerts |
| `METRICS_CHANNEL_ID` | | Global owner channel for weekly metrics |
| `NOTIFY_USER_ID` | | User to @mention in error alerts |
| `MAX_RESPONSE_TOKENS` | `512` | Max tokens per response (admins get 4096 for multi-step plans) |
| `PERSONALITY_DIGEST_INTERVAL` | `15` | Interactions between personality digests |
| `MAX_HISTORY_MESSAGES` | `6` | Context messages sent to API per exchange |
| `RATE_LIMIT_USER_SECONDS` | `5` | Cooldown per user |
| `RATE_LIMIT_CHANNEL_COUNT` | `5` | Max messages per channel in time window |
| `HISTORY_RETENTION_DAYS` | `14` | Days to keep conversation history |
| `LOG_LEVEL` | `info` | `debug` / `info` / `warn` / `error` |

## Architecture

```
src/
  ai/
    chat.js              — Main chat logic, model routing, tool execution loop
    adminTools.js        — 23 admin tools with sanitization + confirmation + undo
    userTools.js         — 6 user tools (timezone, set_reminder, recurring, list, cancel, reschedule)
    systemPrompt.js      — Dynamic system prompt with user context (tz, current local time, delivery prefs)
    personality.js       — Personality digest system (Haiku-powered)
    client.js            — Anthropic SDK client
  db/
    init.js              — SQLite setup (WAL mode, 11 tables) + idempotent v1.2 migration
    credits.js           — Per-guild credit ledger (lifetime/spent/owner_managed)
    history.js           — Conversation storage + channel memory
    personality.js       — Guild personality CRUD
    reminders.js         — Reminder CRUD with ownership enforcement
    tokenBudget.js       — Per-call usage recording, delegates to credits
    userSettings.js      — Per-user timezone + secretary preferences
    guildChannels.js     — Per-guild error/metrics/notices channel routing
  commands/               — Slash command handlers (8 active commands)
  events/                 — Discord event handlers (ready, messageCreate, interactionCreate)
  utils/
    rateLimiter.js       — Per-user + per-channel rate limiting
    adminRateLimiter.js  — Per-guild admin tool rate limiting (10/60s)
    reminderScheduler.js — Restart-safe reminder timers + hourly sweep
    secretaryScheduler.js— 5-min poll for daily digest delivery
    updateNotifier.js    — Version-bump announcements (idempotent, prod-only)
    aiSuggestionButton.js— ✨ AI-suggested reminder title button + listener
    timezone.js          — IANA tz helpers + Discord auto-timestamp formatter
    backup.js            — Daily DB backup with retention
    weeklyMetrics.js     — Owner global + per-guild weekly reports
    errorNotifier.js     — Error alerts (global + per-guild fan-out)
    logger.js            — Structured JSON logging
  version.js              — BOT_VERSION + per-version CHANGELOG
  config.js               — Frozen config from env vars
  index.js                — Entry point, scheduler init, graceful shutdown
tests/
  budget.test.js          — 40 per-guild credit tests (in-memory DB)
  adminSanitization.test.js — 58 admin tool input validation tests
tools/
  fire-test-notice.js     — Dev tool to preview update notices
```
